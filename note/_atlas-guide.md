## Context Note Wiki — Atlas 활용 가이드

이 위키는 LLM(Atlas)에게 구조화된 컨텍스트를 전달하기 위한 시스템입니다.
아래 프롬프트를 Atlas Side Chat에 붙여넣어 사용하세요.

## 컨텍스트 주입 프롬프트

### 1. 위키 구조 인식

아래 프롬프트로 Atlas에게 현재 위키 구조를 인식시킵니다.

```
당신은 Context Note Wiki의 Atlas입니다.
이 위키는 파일 트리 = 컨텍스트 구조, 파일 = 컨텍스트 단위입니다.
모든 문서는 다음 섹션 구조를 따릅니다:

- Problem: 해결하려는 문제 정의
- Design: 설계 방향과 제약 조건
- Dev: 구현 세부사항
- Test: 검증 기준과 결과
- Decision: 확정된 의사결정 (필수)
- Next: 다음 단계 액션

규칙:
1. 구조 > 내용 (섹션 구조를 항상 유지할 것)
2. Decision 섹션은 반드시 포함할 것
3. 하나의 파일 = 하나의 주제 (컨텍스트 최소화)
4. 분석 결과는 해당 섹션에 직접 반영할 것
```

### 2. 문서 분석 요청

특정 문서를 Atlas에게 분석시킬 때 사용합니다.

```
아래는 [주제명] 컨텍스트 노트입니다.

[여기에 문서 내용 붙여넣기]

다음을 수행하세요:
1. Problem이 명확히 정의되어 있는지 평가
2. Design이 Problem을 해결하는지 검증
3. Decision이 누락되었으면 근거 기반으로 제안
4. Next에 실행 가능한 액션이 있는지 확인
```

### 3. Decision 도출

의사결정이 필요할 때 Atlas에게 판단을 요청합니다.

```
아래 컨텍스트를 기반으로 Decision을 도출하세요.

[문서 내용 붙여넣기]

출력 형식:
- 결정사항: (한 줄 요약)
- 근거: (판단 이유 2-3개)
- 트레이드오프: (선택하지 않은 대안과 그 이유)
- 리스크: (이 결정의 잠재적 위험)
```

### 4. 컨텍스트 정리 요청

기존 문서의 내용이 산만할 때 구조화를 요청합니다.

```
아래 내용을 Context Note 형식으로 정리하세요.

[자유 형식 텍스트 붙여넣기]

규칙:
- 반드시 Problem / Design / Dev / Test / Decision / Next 섹션으로 분류
- 분류할 수 없는 내용은 가장 가까운 섹션에 배치
- Decision이 없으면 "미결정 — 추가 논의 필요" 명시
- 각 섹션은 불릿 리스트로 핵심만 기술
```

### 5. 멀티 파일 비교 분석

여러 컨텍스트 노트를 비교할 때 사용합니다.

```
아래는 동일 프로젝트의 컨텍스트 노트 2개입니다.

[파일 A 내용]
---
[파일 B 내용]

다음을 분석하세요:
1. 두 문서 간 모순되는 Decision이 있는가
2. 중복되는 Problem이 있는가
3. 통합 가능한 내용이 있는가
4. 각 문서의 Next가 서로 의존하는가
```

### 6. 새 컨텍스트 생성 요청

새로운 주제를 시작할 때 Atlas에게 초안을 요청합니다.

```
[주제 설명]에 대한 Context Note 초안을 작성하세요.

출력 형식:
## Problem
(이 주제에서 해결할 핵심 문제)

## Design
(접근 방식과 제약 조건)

## Dev
(구현 시 고려사항)

## Test
(검증 기준)

## Decision
(현재 확정된 사항, 없으면 "미결정")

## Next
(즉시 실행 가능한 다음 단계)
```

## Decision

- 이 가이드는 Atlas 활용의 표준 프롬프트 세트로 사용한다
- 프롬프트는 복사-붙여넣기로 즉시 사용 가능해야 한다
- 위키 구조(6개 섹션)를 Atlas가 항상 인식하도록 세션 시작 시 "위키 구조 인식" 프롬프트를 먼저 주입한다
