## Problem

LLM과 대화할 때 컨텍스트가 누적되면서 핵심이 희석된다. 긴 대화에서 이전 결정사항을 놓치거나 같은 논의를 반복하게 된다.

## Design

- 하나의 파일 = 하나의 주제 (컨텍스트 최소화)
- 파일 트리로 주제 간 관계를 구조화
- 각 문서는 Problem → Decision 흐름을 강제

## Decision

- Context Note는 "기록"이 아니라 "의사결정 자산"이다
- 모든 문서는 반드시 Decision 섹션을 포함한다
- 구조 없는 자유 텍스트는 허용하지 않는다
